# 🚀 基于微服务架构的外卖全链路后台管理系统

本项目是一个基于微服务架构的外卖全链路后台管理系统，采用**前后端分离**开发模式，全面覆盖**用户端（微信小程序）**与**管理端（PC后台）**两大交互终端。

---

## 🚀 快速开始

### 📋 环境要求
* **后端**：JDK 17+ 、Maven 3.8+
* **数据库**：MySQL 8.0 、Redis 7.0
* **前端**：Node.js 18+
* **工具**：微信开发者工具

### 💻 本地运行指南

#### 1. 初始化数据库
在 MySQL 客户端中创建数据库并执行初始化脚本：
```sql
source take-out.sql

```

#### 2. 修改配置文件

打开后端开发环境配置文件，修改您的数据库与 Redis 连接地址：

* **路径**：`springboot/server/src/main/resources/application-dev.yml`

#### 3. 启动后端服务

```bash
cd springboot/server
mvn clean install
mvn spring-boot:run

```

#### 4. 启动管理端 (PC)

```bash
cd vue3
npm install
npm run dev

```

#### 5. 启动小程序端 (用户端)

1. 打开 **微信开发者工具**。
2. 导入项目，选择目录为项目的 `app/` 文件夹。

## ✨ 核心功能架构

### 📱 用户端（微信小程序）
* **身份认证**：支持微信授权一键登录。
* **商品浏览**：分类清晰浏览菜品与超值套餐。
* **购物车管理**：灵活进行商品增删改查、自动计算总价。
* **精细化下单**：支持口味选择、多地址选择、订单备注、餐具数量选择。
* **支付环节**：安全模拟微信支付流程。
* **订单追踪**：全流程历史订单追踪（*待接单 → 配送中 → 已完成*）。
* **地址簿**：提供完整的地址 CRUD 管理，支持设置默认地址。

### 💻 管理端（PC 后台）
* **权限分级**：员工登录/注册，严格划分**超级管理员**与**普通员工**权限。
* **运营仪表盘**：全方位营业数据概览，集成 ECharts 动态图表。
* **分类管理**：支持菜品分类与套餐分类的灵活配置。
* **核心 CRUD**：
    * **菜品管理**：支持自定义口味配置、批量启售/停售。
    * **套餐管理**：支持多菜品关联组合、灵活定价。
* **订单动态调度**：支持接单、拒单、取消、派送、完成的全生命周期管理。
* **数据统计与分析**：
    * 营业额趋势、用户增长趋势、订单量统计、销量 Top 10 榜单。
    * 支持报表一键导出至 Excel。
* **实时提醒**：基于 WebSocket 的新订单语音及弹窗实时通知。

## 🔒 安全机制与架构设计

| 安全维度/机制 | 实现方式与应用场景 |
| :--- | :--- |
| **认证机制** | **JWT 双 Token 验证**：管理员与用户端采用独立密钥，保障身份安全。 |
| **授权机制** | **权限分级控制**：区分超级管理员与普通员工，严格限制敏感操作。 |
| **密码保护** | **MD5 哈希加密**：数据库敏感密码洗白存储，防止拖库泄露。 |
| **上下文隔离** | **ThreadLocal 线程隔离**：安全存储当前请求的用户 ID，防止并发越权。 |
| **全局异常捕获** | **GlobalExceptionHandler**：统一拦截系统异常，避免暴露后端堆栈信息。 |
| **公共字段填充** | **AOP 切面编程**：自动拦截并填充 `create_time`、`update_time` 等公共字段。 |
| **自动化任务** | **Spring Task 定时任务**：15分钟未支付自动取消订单、1小时派送超时自动完成。 |

## 🎨 设计模式应用实战

> 项目在核心代码架构中深度沉淀了设计模式，以保障代码的**可扩展性**与**高内聚低耦合**：

* **单例模式**：应用于 `WebSocketServer` 会话管理，确保全局单一连接控制器。
* **观察者模式**：应用于 **WebSocket 订单状态实时推送**，状态变更自动通知前端。
* **代理模式**：应用于 `AutoFillAspect` **AOP 自动填充**，无感拦截并增强方法。
* **策略模式**：应用于 **JWT 双 Token 认证拦截器**，根据不同终端动态匹配校验策略。
* **模板方法模式**：继承并扩展 `WebMvcConfiguration` 框架模板，统一配置拦截器与资源映射。
* **工厂模式**：通过 `@Bean` 配置类集中管理、生产并注入核心组件。
